我们构建一个项目，会使用许多第三方依赖，此时为了更好地管理依赖和开发项目，会使用包管理器，在 JS 中，我们通常使用 npm、pnpm、yarn 等。

在此项目中，使用 npm，使用 npm init 可以将一个文件夹变为一个 npm 项目，会在根目录下生成一个 package.json 文件，用于管理依赖、项目元数据、自定义命令等。

我们使用 npm install xxx 进行下载依赖时，对应的依赖会出现在 package.json 中，同时，别人下载了这个项目时，使用 npm install 时，会根据 package.json 下载项目需要的依赖。

当下载了一个项目后使用 npm install --save 会下载 package.json 中 dependencies 中的依赖，而不会下载 devDependencies 中的，使用 npm install --save-dev 会同时下载两个部分的依赖。

packeage.json 文件中的 script 属性中可以配置命令的简化版，比如：配置一个 "lint": "eslint --quiet --ext js,vue ."，在终端就可以使用 npm run lint 来调用 "lint" 对应的命令，非常简便。
